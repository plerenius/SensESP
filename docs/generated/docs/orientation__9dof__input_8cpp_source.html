43 // Provides one parameter reading from the combination sensor.
44 // val_type determines which particular parameter is output.
45 // Note that since the filter algorithm works best when all 9-DOF
46 // parameters are captured simultaneously, and at a specified
47 // constant rate, there is only a single call to
48 // sensor_fxos_fxas->gatherOrientationDataOnce() in this method.
49 // Arbitrarily, this update happens when the compass heading
50 // is requested; all the remaining parameters are retrieved
51 // by calling the appropriate sensor_fxos_fxas->getter method.
52 // If the compass heading parameter is not the one that is set
53 // to the fastest repetition rate (in main.cpp when calling
54 // Read9DOF::Read9DOF()), then you should move the
55 // gatherOrientationDataOnce() call into the case block
56 // for whichever parameter _is_ updated the fastest - this
57 // ensures that that parameter and all others are never stale.
58 void Read9DOF::update() {
